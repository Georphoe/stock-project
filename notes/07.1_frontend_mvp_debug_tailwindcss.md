二、幫你整理好的 Debug 筆記（精簡 + 工程感）

🧩 07.1_frontend_mvp_debug_tailwindcss.md
📅 日期：2026-03-21
💡 主題：Tailwind 無法解析（Module Resolution Debug）

一、問題現象

啟動前端專案時出現錯誤：

Error: Can't resolve 'tailwindcss'
錯誤特徵
Webpack 找不到 tailwindcss
module resolution 失敗
node_modules 解析路徑異常

二、問題本質

👉 Node module resolution 跑錯層級

Node 解析順序：

當前目錄 node_modules
上一層 node_modules
更上層
🚨 關鍵問題

專案出現：

stock-project/
├── package.json   ❌（錯誤）
├── node_modules/  ❌（錯誤）
└── frontend/
    ├── package.json ✅

👉 導致：

Webpack 從 root 開始找 module
root 沒有 tailwindcss
→ 解析失敗

三、根本原因（Root Cause）

👉 在錯誤目錄執行了：

npm install

或

npm init

導致：

root 被誤當成 Node 專案
產生：
package.json
node_modules
package-lock.json

📌 這是污染專案結構的典型問題

四、Debug 過程
1️⃣ 觀察錯誤訊息
確認是 module resolution 問題
指向 tailwindcss
2️⃣ 檢查專案結構
發現 root 有 package.json
→ 非預期結構
3️⃣ 檢查依賴位置
frontend 有 node_modules
root 也有 node_modules（異常）

👉 依賴來源混亂

4️⃣ 分析 module resolution
Webpack 從 root 開始解析
不是從 frontend
5️⃣ rollback
回到上一次正常 commit
6️⃣ 發現 untracked files
package.json
node_modules/

👉 這是關鍵線索：

Git 沒追蹤
但實際存在
7️⃣ 定位問題

👉 結論：

root 被誤建為 Node 專案

五、解決方法
✅ 1. 刪除 root 錯誤 Node 專案
package.json
node_modules/
package-lock.json
✅ 2. 保留 frontend 為唯一專案
frontend/
  package.json
  node_modules/
✅ 3. 重新安裝依賴
npm install
✅ 4. 安裝 tailwind
npm install tailwindcss postcss autoprefixer
✅ 5. 正確啟動
cd frontend
npm run dev

六、核心觀念
🧠 1. Node module resolution
會往上找 node_modules
任一層 package.json 都會影響解析
🧠 2. npm 會污染目錄

👉 在錯誤目錄執行 npm install
= 會生成新的 Node 專案

🧠 3. untracked files 是關鍵線索
可以發現「非 Git 管控的異常」
🧠 4. 專案結構一致性

👉 一個 frontend 專案應該只有：

一個 package.json
一個 node_modules

七、一句話總結（面試用）

這次問題是因為在專案根目錄誤建立 Node 專案，導致 module resolution 從錯誤層級解析，進而無法找到 tailwindcss。透過 rollback 與 untracked files 發現異常，刪除 root 依賴後恢復正常。
